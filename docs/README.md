# Documentação de projeto

Nessa pasta espera-se a documentação geral do projeto relativa a temas como orçamento, organização da equipe, relatórios.

> [!NOTE]
> Os arquivos Markdown (`.md`) nesta pasta (especificamente nas subpastas `energia/`, `software/`, `estrutura/` e `hardware/`) são utilizados para gerar automaticamente o site de documentação do projeto hospedado no **GitHub Pages** (via MkDocs).

## Estrutura da Pasta

- `docs/energia/`: Documentação relacionada ao subsistema de energia.
- `docs/hardware/`: Documentação do hardware do projeto.
- `docs/software/`: Documentação do software e firmware.
- `docs/estrutura/`: Documentação da estrutura mecânica.
- `docs/relatorio/`: Pasta com os arquivos do relatório técnico (LaTeX).

---
Para escrita do relatório em LaTeX, recomenda-se o VS Code com a extensão [LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop).

> [!WARNING]
> Os arquivos .pdf ou arquivos intermediários de compilação do LaTeX não devem ser comitados.
